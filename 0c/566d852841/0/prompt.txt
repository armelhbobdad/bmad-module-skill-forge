# SKF Knowledge Folder — Implementation Plan

## Context

The SKF module has 10 workflows with 19 operational `data/` files embedded in workflow step directories, but **no agent-level knowledge folder**. The TEA reference module demonstrates the pattern: a `knowledge/` folder with standalone reference documents that the agent consults just-in-time during workflow execution, indexed via a CSV file.

Knowledge files are **cross-cutting principles and patterns** that Ferris needs across multiple workflows — distinct from workflow `data/` files which are step-specific operational artifacts (schemas, templates, scoring formulas). We also incorporate the agentskills.io specification and Claude best practices as authoritative references for the output format SKF produces.

## Implementation

### Phase 1: Create directory and index

**Create** `src/knowledge/` directory and `src/knowledge/skf-knowledge-index.csv`

CSV format follows TEA pattern (`tea-index.csv`):
```
id,name,description,tags,tier,fragment_file
```

8 entries mapping to the knowledge files below. All tagged `core` tier (these are foundational, not optional).

### Phase 2: Create 8 knowledge files

All files follow the TEA structural pattern:
- `# Title` (H1)
- `## Principle` — one-paragraph core concept
- `## Rationale` — problem/solution framing
- `## Pattern Examples` — concrete examples with `### Example N: Title`, **Context**, **Implementation**, **Key Points**
- Optional: `## Anti-Patterns`, `## Checklist`, `## Integration Points`
- `_Source:` attribution line
